# Contributing to OpenSTE

Thank you for your interest in contributing to OpenSTE.

This project welcomes contributions from technical writers, developers,
and documentation specialists.

## Ways to Contribute

You can contribute in many ways:

-   Improve writing guidelines
-   Suggest vocabulary additions
-   Add documentation examples
-   Improve project documentation
-   Develop validation or linting tools
-   Fix issues

## Contribution Workflow

1.  Fork the repository
2.  Create a new branch
3.  Make your changes
4.  Commit your changes
5.  Push your branch
6.  Open a Pull Request

Example:

``` bash
git checkout -b feature/improve-guidelines
git commit -m "Improve guideline clarity"
git push origin feature/improve-guidelines
```

## Guidelines for Contributions

-   Keep writing clear and concise
-   Follow OpenSTE writing rules
-   Provide examples where possible
-   Use consistent terminology

## Pull Request Process

Pull requests should:

-   Describe the change clearly
-   Reference related issues if applicable
-   Include examples when modifying writing rules
-   Pass validation tools if available

All contributions are reviewed by project maintainers before merging.

## Code of Conduct

By participating in this project, you agree to follow the project's Code
of Conduct.
