# Documentation

The **docs** directory contains detailed documentation about the OpenSTE
project.

This documentation explains the structure, philosophy, and usage of
OpenSTE.

------------------------------------------------------------------------

## Purpose

The documentation helps users understand:

-   how OpenSTE works
-   how to apply the writing rules
-   how to use the vocabulary system
-   how to contribute to the project

------------------------------------------------------------------------

## Contents

Typical documentation may include:

    specification.md
    writing_guide.md
    project_overview.md
    contributor_guide.md

------------------------------------------------------------------------

## Topics

Documentation may cover:

-   writing methodology
-   vocabulary management
-   documentation standards
-   tooling and integrations
-   project governance

------------------------------------------------------------------------

## Audience

The documentation is intended for:

-   technical writers
-   documentation teams
-   developers
-   translators
-   organizations adopting OpenSTE

------------------------------------------------------------------------

## Contributing

You can contribute by:

-   improving explanations
-   expanding documentation
-   adding tutorials
-   documenting tools and workflows

Clear documentation helps the community adopt and use OpenSTE
effectively.
