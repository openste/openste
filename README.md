**OpenSTE** is an open-source initiative that provides a clear and
structured approach to simplified technical writing.

The goal of the project is to help organizations and individuals create
documentation that is **easy to read, consistent, and easier to
translate**. OpenSTE focuses on clear writing rules, a controlled
vocabulary, and community collaboration.

This project is **open, transparent, and community-driven**, allowing
anyone to contribute and improve the standard.

## Overview

Technical documentation can become difficult to read due to:

-   complex sentences
-   inconsistent terminology
-   unclear instructions
-   translation challenges

OpenSTE addresses these problems by providing:

-   simple writing rules
-   controlled vocabulary
-   consistent sentence structures
-   documentation examples
-   open collaboration

The goal is to make documentation understandable for both **native and
non‑native English readers**.

------------------------------------------------------------------------

## Why OpenSTE

OpenSTE focuses on improving documentation quality while remaining fully
open.

Benefits include:

-   **Clearer instructions**
-   **Reduced ambiguity**
-   **Improved translation**
-   **Consistent terminology**
-   **Community improvements**

The project allows teams to adopt a structured writing approach without
licensing restrictions.

------------------------------------------------------------------------

## Core Principles

### 1. Clarity

Write sentences that are direct and easy to understand.

------------------------------------------------------------------------

### 2. Consistency

Use the same words for the same concepts across all documentation.

------------------------------------------------------------------------

### 3. Simplicity

Prefer short sentences and simple grammar.

------------------------------------------------------------------------

### 4. Structured Instructions

Write instructions in clear steps.

## Getting Started

Clone the repository:

``` bash
git clone https://github.com/openste/openste.git
```

Navigate to the project:

``` bash
cd openste
```

Review the writing rules:

    guidelines/

Review the vocabulary list:

    vocabulary/

Start applying the rules to your documentation.

## Contributing

Contributions are welcome from:

-   technical writers
-   developers
-   documentation specialists
-   translators
-   educators

Ways to contribute:

-   improve writing rules
-   add vocabulary terms
-   create documentation examples
-   build tooling or integrations
-   fix issues

Typical workflow:

``` bash
git checkout -b feature/improve-guidelines
git commit -m "Improve writing guideline clarity"
git push origin feature/improve-guidelines
```

Then submit a Pull Request.
