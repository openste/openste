# Guidelines

The **guidelines** directory contains the core writing rules used by
OpenSTE.\
These rules help writers produce technical documentation that is clear,
consistent, and easy to understand.

The purpose of the guidelines is to standardize how technical content is
written so that readers can quickly understand instructions, procedures,
and descriptions.

------------------------------------------------------------------------

## Purpose

The writing guidelines aim to:

-   improve clarity in technical documentation
-   reduce ambiguity in instructions
-   simplify sentence structure
-   create consistent documentation across projects
-   make documentation easier to translate

------------------------------------------------------------------------

## Scope

The guidelines cover topics such as:

-   sentence structure
-   grammar rules
-   punctuation usage
-   instruction formatting
-   terminology usage
-   style recommendations

------------------------------------------------------------------------

## Structure

Typical files in this directory may include:

    sentence_rules.md
    grammar_rules.md
    instruction_formatting.md
    punctuation.md
    writing_style.md

Each file focuses on a specific aspect of writing.

------------------------------------------------------------------------

## Example

**Complex sentence**

    Prior to commencing the installation process, verify that the device has been disconnected from the power supply.

**Simplified sentence**

    Disconnect the power supply before you start the installation.
